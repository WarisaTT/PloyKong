Nginx — Reverse Proxy + Wildcard Domain

NGINX
1

เข้าใจ Routing Architecture

routing 
# Traffic flow:
app.ploykong.com        → Vue.js Frontend (Vercel หรือ Nginx static)
api.ploykong.com        → Go Fiber API (Docker container :3000)
napat.ploykong.com      → Go API renders portfolio by slug "napat"
alice.ploykong.com      → Go API renders portfolio by slug "alice"
*.ploykong.com          → Wildcard → Go API + X-Portfolio-Slug header
2

Start Nginx Container

bash 
# รัน nginx ด้วย docker compose (ดู docker-compose.yml)
docker compose up -d nginx

# ทดสอบ config
docker exec ploykong-nginx nginx -t
# nginx: the configuration file /etc/nginx/nginx.conf syntax is ok
# nginx: configuration file /etc/nginx/nginx.conf test is successful

# Reload nginx
docker exec ploykong-nginx nginx -s reload

CI/CD Pipeline — GitHub Actions Auto Deploy

CI/CD
1

ตั้งค่า GitHub Secrets

ไปที่ GitHub Repository → Settings → Secrets and variables → Actions → New repository secret

secrets 
VPS_HOST         = 203.0.113.10          # IP ของ VPS
VPS_USER         = root                   # หรือ deploy user
VPS_SSH_KEY      = (SSH private key)      # cat ~/.ssh/id_rsa
DB_PASSWORD      = your-db-password
JWT_SECRET       = your-jwt-secret-64-chars
OPENAI_API_KEY   = sk-your-openai-key
MYSQL_ROOT_PASSWORD = root-password
2

ตั้งค่า SSH Access บน VPS

bash 
# บน Local Machine — สร้าง SSH key สำหรับ GitHub Actions
ssh-keygen -t ed25519 -C "github-actions" -f ~/.ssh/ploykong_deploy

# Copy public key ไปยัง VPS
ssh-copy-id -i ~/.ssh/ploykong_deploy.pub root@YOUR_VPS_IP

# Copy private key เป็น GitHub Secret VPS_SSH_KEY
cat ~/.ssh/ploykong_deploy
